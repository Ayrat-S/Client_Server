   Что такое клиент серверная архитектура?

   КЛИЕНТ
   Клиент - это аппаратный или программный компонент вычислительной системы, посылающий запросы серверу.
   
   СЕРВЕР
   Сервер - это программный компонент вычислительной системы, выполняющий сервисные (обслуживающие) функции по запросу клиента, предоставляющий ему доступ к определенным ресурсам или услугам.

   КЛИЕНТ-СЕРВЕР
   Фактически клиент-сервер - это программное обеспечение, т.е. вычислительная или сетевая архитекура, в которой задания или сетевая нагрузка распределена между поставщиками услуг, называемыми серварми, и заказчиками услуг, называемыми клиентами. 
Обычно эти программы расположены на разных вычислительных машинах и взаимодействуют между собой через вычислительную сеть посредством сетевых протоколов, независимо друг от друга. 
Но они могут быть расположены и на одной машине.

   СЕТЕВОЙ ПРОТОКОЛ
   Сетевой протокол - это набор правил и действий (очередность действий), позволяющий осуществлять сетевое соединение между двумя и более устройствами для обмена данными между собой.
Наиболее распространённой системой классификации сетевых протоколов является так называемая модель OSI, в соответствии с которой протоколы делятся на 7 уровней по своему назначению — от физического (формирование и распознавание электрических или других сигналов) до прикладного (интерфейс программирования приложений API для передачи информации приложениями).
   
   СЕТЕВАЯ МОДЕЛЬ OSI - Open System Interconnection (взаимодействие открытых систем).
Сетевая модель OSI - сетевая модель стека (магазина) сетевых протоколов OSI/ISO.
Уровни модели OSI:
7. Прикладной (application) - Доступ к сетевым службам.
6. Представления (presentation) - Представление и шифрование данных.
5. Сеансовый (session) - Управление сеансом связи
4. Транспортный (transport) - Прямая связь между конечными пунктами и надёжность.
3. Сетевой (network) - Определение маршрута и логическая адресация.
2. Канальный (data link) - Физическая адресация.
1. Физический (physical) - Работа со средой передачи, сигналами и двоичными данными.

   API - Application Programming Interface
   Программный интерфейс приложения - описание способов (набор классов, процедур, функций, структур или констант), с помощью которых одна программа может взаимодействовать с другой программой.
